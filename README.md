# rc-vs-sf

リザバー計算(ESN + RLS/LMS オンライン学習)と Dogariu–Brahmbhatt–Hazan
「Universal Learning of Nonlinear Dynamics」(arXiv:2508.11990)のスペクトラル
フィルタリングを、公平なオンライン設定(逐次予測 → 真値観測 → 逐次更新)で
比較する実験プロジェクト。

## セットアップ

```bash
uv sync
uv run pytest            # 全テスト
uv run python experiments/reproduce_paper.py   # 論文再現ゲート
```

## 手法

| 名前 | 内容 |
|---|---|
| sf / sf+obs | 論文 Algorithm 1(観測スペクトラルフィルタリング)。torch + COCOB(parameterfree) |
| sfedmd | eDMDリフト空間でのスペクトラルフィルタ特徴 + リッジ回帰(T/10ごと再フィット) |
| edmd | PyKoopman eDMD(恒等 + thin-plate RBF 20個、T/10ごと再フィット) |
| esn-rls / esn-lms | ESN(ReservoirPy)+ RLS / LMS オンラインリードアウト |
| persistence | ŷ_t = y_{t-1}(全プロットの床) |

## 論文からの逸脱・曖昧さの解決(重要)

論文の図(arxiv.org/html/2508.11990v1/figs/)と直接照合して以下を確定した。

1. **軌道長**: 論文の "T=1024" は Hankel フィルタの地平線であり軌道長ではない。
   図の x 軸は Lorenz/振り子/Langevin で ~10⁴、置換LDSで 5×10⁴、Gaussian LDS で
   5×10³。SFeDMD の誤差が落ちる時刻が t=1000(=T/10 の初回再フィット)である
   ことからも確認できる。
2. **uフィルタ窓のラグ**: 論文の式は入力フィルタ窓を `u_{t-2:t-T}` と書くが、
   実験設定の m=1 では入力ARブロックが空になり、lag-2 開始だと u_{t-1} の情報が
   完全に欠落して1ステップのLDSすら学習不能になる。本実装は窓を lag-1
   (`u_{t-1}` から)に揃えた。y窓は論文どおり lag-1 から、長さ L = min(T-1, 1024)。
   検証: 同一特徴量でのオフラインリッジが Gaussian LDS で誤差 ~0、置換LDSで
   0.009 に達する(=特徴量構築は十分)。
3. **置換LDSは自律系**: u 駆動だと限界安定系の状態分散が t に比例して成長し、
   論文の図の平坦な vanilla 曲線と矛盾する。x0 の周期16回転(u=0)が図と整合。
4. **観測の標準化**: 論文の損失スケールが O(1) であることから、観測は標準化
   されていると判断。比較実験では検証シードの統計で全手法共通に標準化(因果的)。
5. **学習損失とオプティマイザ**: Algorithm 1 の損失は非二乗ノルムだが、二乗損失の
   方が COCOB で約2倍速く収束し論文の曲線に近い。さらに線形系では COCOB でも論文の
   図より数倍遅く、`optimizer="rls"`(二乗損失の線形予測器ではONSと等価、論文も
   ONS を代替に挙げる)で論文の図を再現(置換: SF+obs 0.03 / vanilla 0.75 停滞、
   Gaussian: SF+obs 床 3e-6 ≪ vanilla 床 1.4e-3)。COCOB は parameterfree
   (原著者実装)を使用。RLS-SF は ESN の RLS リードアウトと同じ学習器なので、
   比較が「同じオンライン学習器、異なる固定特徴量」という構図になる。
6. **線形系1のノイズ周波数**: 論文の `w_t = 0.1 sin(3πt)·1` は整数 t で恒等的に 0
   (一方、図には明確なノイズ床がある)。t を連続時間 k·dt (dt=0.01) と読み、
   ω = 3π/100(周期~67ステップ)をデフォルトとした。
7. **Langevinの結合定数**: 論文の `c=0.2, d=64` ではポテンシャルが対角方向に
   非有界(0.05Σx⁴ を結合4次項が支配)で Euler–Maruyama が即発散する。
   有界性条件 c ≤ 0.1/(d-1) に余裕を持たせ、デフォルト c = 0.05/(d-1) とした。
8. **二重振り子**: 論文は Brax のカート付き二重振り子 + 線形フィードバック制御。
   本実装は古典的な自律二重振り子ODE(RK4)で代替し、観測は Brax 流の
   [sinθ₁, cosθ₁, sinθ₂, cosθ₂, ω₁, ω₂](d=6)。
9. **eDMDの数値設定**: pykoopman のデフォルト回帰器(PyDMDRegressor、SVD打ち切り)
   は精度が桁で悪いため `regression.EDMD()` を明示。thin-plate RBF は生スケールで
   破綻するため、履歴統計による標準化(因果的)を挟む。
10. **SFeDMDの数値設定**: σ^{1/4} スケーリングで特徴量列のスケールが桁違いに
    なるため、リッジ回帰前に列標準化。初期過渡(Langevin は x0=0 から ±6σ)が
    回帰を汚染するため再フィット時に序盤 washout を除外。射影(Algorithm 1 の
    クリップ)の半径は観測ノルムの95パーセンタイルの2倍(最大値基準だと過渡で
    半径が一桁膨らみ射影が無効化される)。Langevin のようなノイズ支配系では
    リッジ強め(1e-1)が必要(グリッドサーチ対象)。
11. **依存関係**: reservoirpy は 0.3.16 に固定(0.4.x は scipy≥1.12 を実質要求、
    pykoopman 1.2.1 は scipy≤1.11.2 を要求するため共存不可)。

## 公平性プロトコル

- 同一 (系, シード) の軌道を全手法に再生
- predict は y_t にアクセス不可(テストで監査)、washout 100 ステップ共通
- ハイパーパラメータは検証シード(3本)で選び、テストシード(12本)は1回だけ実行
